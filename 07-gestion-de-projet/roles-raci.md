# Rôles d'équipe & matrice RACI

**Contexte :** équipe cross-villes de 3 membres ("EPI'Gency Digital"), en autonomie complète (pas de bootstrap), en compétition collaborative avec d'autres équipes évaluées par les pairs.

## Rôles proposés

Sur un format aussi court (3 semaines) et une équipe de 3 personnes, les rôles sont **portés en responsabilité principale, pas en exclusivité** — chacun contribue à tout, mais un référent garantit la cohérence :

| Rôle | Référent | Périmètre |
|---|---|---|
| **Product Owner / Recherche UX** | Membre A | Vision produit, personas, priorisation des besoins, conformité juridique |
| **Design / Prototypage** | Membre B | Design system, wireframes, prototype haute-fidélité, cohérence visuelle |
| **Gestion de projet / Qualité** | Membre C | Planning, versioning Git, revue de conformité aux consignes, préparation de la soutenance |

## Matrice RACI (R = Réalise, A = Approuve, C = Consulté, I = Informé)

| Livrable | Membre A | Membre B | Membre C |
|---|---|---|---|
| Audit UX Parcoursup | R/A | C | C |
| Personas & parcours | R | C | A |
| Benchmark concurrentiel | C | R | A |
| Vision produit & user stories | R/A | C | C |
| Design system | C | R/A | I |
| Wireframes basse fidélité | C | R | A |
| Prototype haute fidélité | I | R/A | C |
| Conformité RGPD / juridique | R/A | I | C |
| Plan de conduite du changement | R | C | A |
| Planning & versioning | I | C | R/A |
| Support de soutenance | C | C | R/A |

## Répartition de la prise de parole en soutenance

Pour éviter qu'un seul membre porte toute la présentation (critère de peer evaluation mentionné dans le kick-off), chaque membre présente la partie dont il est référent : recherche/vision (Membre A), démonstration du prototype (Membre B), méthodologie/organisation/conformité (Membre C).
