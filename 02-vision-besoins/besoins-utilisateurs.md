# Définition des besoins — User stories

Format : *En tant que [persona], je veux [action], afin de [bénéfice].*
Priorisation MoSCoW pour le périmètre maquettes (Must / Should / Could / Won't à ce stade).

## Epic Predict — Candidat

| # | User story | Priorité |
|---|---|---|
| P-01 | En tant que Léa, je veux importer mon bulletin scolaire (photo/PDF), afin de ne pas ressaisir manuellement toutes mes notes | Must |
| P-02 | En tant que Léa, je veux voir mon profil de compétences consolidé, afin de comprendre mes points forts et mes points faibles | Must |
| P-03 | En tant que Léa, je veux rechercher des formations et voir un score de correspondance pour chacune, afin de prioriser mes vœux | Must |
| P-04 | En tant que Léa, je veux comprendre le détail du calcul du score (quelles compétences comptent, avec quel poids), afin de ne pas subir une décision de boîte noire | Must |
| P-05 | En tant que Yanis, je veux déclarer des compétences pratiques avec justificatif (projet, stage, certification), afin qu'elles soient prises en compte au-delà de mes notes | Must |
| P-06 | En tant que Léa, je veux comparer jusqu'à 3 formations côte à côte, afin d'arbitrer entre plusieurs choix | Should |
| P-07 | En tant que Léa, je veux accéder à un lexique en langage clair pour les termes techniques, afin de comprendre sans aide extérieure | Should |
| P-08 | En tant que Léa, je veux voir des exemples de profils similaires ayant réussi dans une filière où je suis sous-représentée, afin de lutter contre mon autocensure | Could |
| P-09 | En tant que Léa, je veux utiliser l'outil aussi bien sur mobile que sur ordinateur, afin de l'utiliser où que je sois | Must |

## Epic Prepare — Formation

| # | User story | Priorité |
|---|---|---|
| F-01 | En tant que Camille, je veux créer le référentiel de compétences de ma formation à partir d'un modèle prédéfini, afin de ne pas repartir d'une page blanche | Must |
| F-02 | En tant que Camille, je veux pondérer chaque compétence en %, afin d'objectiver les critères de sélection avec ma commission | Must |
| F-03 | En tant que Camille, je veux prévisualiser exactement ce que verra un candidat, afin de vérifier la clarté de mes attendus avant publication | Must |
| F-04 | En tant que Camille, je veux dupliquer les attendus de l'année précédente et les ajuster, afin de gagner du temps chaque campagne | Should |
| F-05 | En tant que Camille, je veux inviter les autres membres de ma commission à collaborer sur le référentiel, afin d'obtenir un consensus tracé | Should |
| F-06 | En tant que Camille, je veux un tableau de bord indiquant le nombre de profils simulés éligibles à ma formation, afin d'anticiper le volume de candidatures | Could |
| F-07 | En tant que Camille, je veux être guidée sur mes obligations légales de publication des critères, afin d'être en conformité sans expertise juridique | Should |

## Hors périmètre explicite pour ce jalon (Won't — semaines 1-3)

- Authentification réelle multi-établissements (SSO académique).
- Connexion à une base de données de production ou à un SI existant.
- Notifications automatiques par SMS/email.
- Module d'administration Ministère (statistiques nationales agrégées) — pertinent en phase Beta (semaine 4+, cf. kick-off).
